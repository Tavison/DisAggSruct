template<typename T, std::size_t N = 0>
std::size_t DisAgg::detail::count_fields ( )
constexpr

Recursively determines the number of fields in aggregate T.

The cascade works by attempting to construct T with N+1 any_type arguments. If that succeeds, the field count is at least N+1, so the function recurses. When adding one more argument fails, the current N is the exact field count.

Because the entire chain is constexpr and only touches unevaluated contexts, there is zero runtime cost.

template<typename T, std::size_t... Is>
bool DisAgg::detail::try_construct_n ( std::index_sequence< Is... > )
constexpr

Tests whether T can be aggregate-initialised with exactly N arguments of type any_type.

Uses a C++20 requires expression to evaluate the validity of

T{ any_type{}, any_type{}, ... } // N times
DisAgg::any_type
Universal probe type that implicitly converts to any type.
Definition DisAggStruct.h:82

and returns true if the expression is well-formed, false otherwise.

Note
The decltype(..., std::true_type{}) / variadic-fallback pattern that works on Clang/GCC triggers a hard error on MSVC when the aggregate initialiser has too many arguments — MSVC does not treat that as a silent SFINAE failure. A requires expression is the correct tool: the standard mandates it returns false rather than propagating the error, and MSVC implements this correctly.

The index sequence Is is used only to repeat any_type{} exactly N times via a comma-operator fold; the actual index values are discarded.
