Detailed Description

Zero-friction strong typedef wrapper.

A Morpheme<Tag, T> is identical to T in every practical sense — it constructs from T implicitly and converts back to T implicitly. All operators are handled by the compiler via that implicit conversion, so Morpheme works with any underlying type, numeric or otherwise.

The sole purpose of the Tag parameter is to make otherwise identical underlying types distinct to the compiler. Two Morpheme types built on the same T but different tags cannot be silently mixed, and the disaggregation layer can dispatch to the correct handler for each, while all other code treats them as plain T values.

Note
Arithmetic on two Morpheme values decays to T (via the implicit conversion), not back to Morpheme. This is the correct trade-off when T may be non-numeric: the tag does its job at the dispatch boundary, and expressions behave exactly like plain T everywhere else.
Defining a named type
// The tag struct can be forward-declared inline in the template argument.
// No macro needed — this is one line of standard C++.
using UUID = DisAgg::Morpheme<struct UUIDTag, int>;
using TermID = DisAgg::Morpheme<struct TermIDTag, int>;
using Label = DisAgg::Morpheme<struct LabelTag, std::string>;
UUID a = 1; // implicit from int
TermID b = 2; // implicit from int
int c = a; // implicit to int

using Name = DisAgg::Morpheme<struct Name##Tag, T>

Declares a named Morpheme type in one line.

Without this macro the equivalent declaration is two lines — a forward declaration of the tag struct followed by the using alias. The macro collapses that into a single readable statement and keeps the tag type name out of user code entirely.

MORPHEME(UUID, int);
MORPHEME(TermID, int);
MORPHEME(Temperature, float);
MORPHEME(Label, std::string);
