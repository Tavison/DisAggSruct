Detailed Description

Use case demonstrations for the sensor subsystem.

Role: End user / caller
This file represents the end user's world. They need to know:
  • The type vocabulary from SensorTypes.h
  • The function name GetSensorData

They do NOT need to know:

  • That DisAgg exists
  • How the back end works
  • How the front end is wired
  • What other structs anyone else has defined

The key point demonstrated here: the end user defines whatever struct expresses exactly what they need. Two fields, eight fields, any order. GetSensorData accepts all of them without modification. The back end never changes. The front end never changes. Only the struct changes, and the struct lives entirely in the caller's code.
