Detailed Description

End-user demonstration of the Weather and Finance domains.

Role: End user
This file only knows about the front-end functions and the struct shapes it cares about. It has no knowledge of back-end implementations, the disaggregation framework, or any other domain.
Domain sealing
WeatherFrontEnd.h and FinanceFrontEnd.h each include only their own back-end header. Putting a finance type inside a weather struct (or vice versa) is a compile error — the offending type is simply never in scope for the other domain's front-end.
