◆ disaggregate()

template<typename T, typename Policy>
T DisAgg::disaggregate ( T s,
Policy policy )

Primary public entry point.

Disaggregates s into a tuple of field references, applies policy to every field, then reconstructs and returns the result as T.

policy is any callable with a generic operator() — typically a lambda that calls a named free function and lets the compiler resolve the correct overload for each field type. The policy knows the function name; it does not know the field types. A field type with no matching overload is a compile error, enforcing handler completeness at build time.

Template Parameters
TAggregate struct type. Deduced from s.
PolicyAny callable: lambda, function object, or std::function.
Parameters
sThe struct to process. Passed by value; the caller's original is never modified.
policyThe callable to invoke on each field.
Returns
A new T with each field populated by its handler.
Example
// Back-end: one free function per type, no framework knowledge needed.
void retrieve(UUID& v) { v = UUID{ db.next_id() }; }
void retrieve(Temperature& v) { v = Temperature{ sensor.read() }; }
// Wiring: lambda names the function; front-end function names the struct.
Reading get_reading() {
return DisAgg::disaggregate(Reading{}, [](auto& f) { retrieve(f); });
}
// Front-end: one call, one return.
Reading r = get_reading();

◆ for_each_field()

template<typename Tuple, typename Fn>
void DisAgg::for_each_field ( Tuple & t,
Fn && fn )

Applies a callable to every element of a tuple.

Visits each element in index order and calls fn on it. The primary use case is building dispatchers that call an overloaded handler function on each struct field without any knowledge of the specific types involved.

Template Parameters
TupleTuple type, typically a reference-tuple from to_tuple.
FnUnary callable. Must be invocable with every element type in t — a missing overload is a compile error by design.
Parameters
tThe tuple to iterate.
fnThe function to call on each element.
Example
// Define one overload per field type — the compiler enforces completeness.
void process(UUID& v) { ... }
void process(ErrorCode& v) { ... }
auto dispatcher = [](auto& t) {
DisAgg::for_each_field(t, [](auto& field) { process(field); });
return t;
};

◆ from_tuple()

template<typename T, typename Tuple>
T DisAgg::from_tuple ( Tuple && t)

Reconstructs an aggregate struct T from a tuple.

Uses std::apply to unpack the tuple elements as arguments to T{...}, which performs aggregate initialisation. Works with both value-tuples and reference-tuples (such as those produced by to_tuple).

Template Parameters
TThe aggregate struct type to construct. Must be specified explicitly since it cannot be deduced from the tuple alone.
TupleAny tuple-like type whose elements are compatible with the fields of T in declaration order.
Parameters
tThe source tuple. Forwarded into the aggregate initialiser, so move semantics are preserved for value-tuples.
Returns
A new instance of T initialised from the tuple elements.
Example
auto t = std::make_tuple(7, 8, 9);
Point p = DisAgg::from_tuple<Point>(t); // p.x=7, p.y=8, p.z=9

◆ to_tuple()

template<typename T>
auto DisAgg::to_tuple ( T & s)

Decomposes an aggregate struct into a std::tuple of lvalue references.

Uses C++17 structured bindings together with an if constexpr ladder keyed on field_count_v<T> to bind all fields by reference and pack them into a std::tie result. Because field_count_v is a compile-time constant, only the matching branch is instantiated — every other branch is discarded by the compiler.

Template Parameters
TAggregate struct type. Deduced from the argument.
Parameters
sThe struct to decompose. Must outlive the returned tuple.
Returns
A std::tuple<F0&, F1&, ...> holding lvalue references to each field of s in declaration order.
Precondition
T must be an aggregate with between 0 and 16 fields (inclusive).
Example
Point p{ 1, 2, 3 };
auto t = DisAgg::to_tuple(p); // tuple<int&, int&, int&>
std::get<0>(t) = 99; // modifies p.x directly

inlineconstexpr

Compile-time field count for aggregate struct T.

Strips cv-qualifiers and references from T before probing, so field_count_v<const Point&> and field_count_v<Point> both work.

Template Parameters
TAny aggregate struct type.
Example
struct Vec3 { float x, y, z; };
static_assert(DisAgg::field_count_v<Vec3> == 3);
